# MemorAId – Microcopy & Content Guidelines

> טקסט הוא UI. כל מילה משפיעה על חוויה, בהירות, ואמון.  
> גרסה: 1.0 | תאריך: פברואר 2026

---

## 1. עקרונות ניסוח – כן / לא

| ❌ לא | ✅ כן |
|-------|-------|
| טכני מדי ("Error 404: resource not found") | ברור ואנושי ("לא מצאנו את הדף המבוקש") |
| פסיבי ("הטופס נשלח") | Active Voice ("שלחנו את הטופס") |
| מאשים ("הזנת נתונים שגויים") | עוזר ("אנא בדוק שוב את הנתונים") |
| מעורפל ("ארעה שגיאה") | ספציפי + פתרון ("לא הצלחנו לשמור. בדוק חיבור לאינטרנט") |
| ארוך מדי ("בשביל להמשיך, יש ללחוץ על הכפתור הבא") | קצר ("המשך") |
| הומור פנימי / משחקי מילים | רציני אך ידידותי |
| יתרונות המוצר ב-error | פתרון בלבד |
| ממוקד מערכת ("המערכת שמרה") | ממוקד משתמש ("הפרטים נשמרו") |

## 2. Button Labels – כללים

### 2.1 עיקרון הפועל (Verb-first)

כל כפתור מתחיל בפועל, מתאר בדיוק מה יקרה:

| ❌ לא | ✅ כן |
|-------|-------|
| "אישור" | "שמור שינויים" |
| "לחץ כאן" | "הוסף מטופל" |
| "שלח בקשה" (כשאין בקשה) | "פתח מדריך" |
| "OK" | "מחק חשבון" |
| "ביטול" לסגירה פשוטה | "סגור" |

### 2.2 פעלים סטנדרטיים לפי פעולה

| פעולה | Label |
|-------|-------|
| שמירת נתונים | "שמור" / "שמור שינויים" |
| יצירת פריט | "הוסף [X]" / "צור [X]" / "הקם [X]" |
| מחיקה | "מחק" (לא "הסר" לפעולות בלתי-הפיכות) |
| ביטול פעולה | "בטל" |
| סגירת dialog | "סגור" (אם לא שומר) |
| ניווט קדימה | "המשך" / "הבא" |
| ניווט אחורה | "חזרה" / "הקודם" |
| חיפוש | "חפש" |
| רענון | "רענן" / "נסה שוב" |
| אישור מחיקה (confirm) | "כן, מחק" |
| ביטול מחיקה | "לא, חזרה" |
| פתיחת תוכן | "פתח מדריך" / "הצג פרטים" |

### 2.3 Loading States – Labels כשרכיב טוען

| מצב | Label |
|-----|-------|
| שמירת נתונים | "שומר..." |
| שליחת טופס | "שולח..." |
| מחיקה | "מוחק..." |
| טעינת נתונים | "טוען..." |
| עיבוד בקשה | "מעבד..." |
| התחברות | "מתחבר..." |

## 3. Error Messages Standard – מבנה הודעת שגיאה

### 3.1 הנוסחה

```
[מה קרה] + [למה / הסבר] + [מה לעשות]
```

**דוגמאות:**

| ❌ לא | ✅ כן |
|-------|-------|
| "שגיאה" | "לא הצלחנו לשמור את הפרטים. בדוק חיבור לאינטרנט ונסה שוב." |
| "Invalid input" | "שדה הטלפון מכיל תווים לא חוקיים. יש להזין ספרות בלבד." |
| "Error 500" | "שגיאת שרת. אנחנו כבר עובדים על זה. נסה שוב בעוד כמה דקות." |
| "Field required" | "שדה זה חובה. יש להזין שם פרטי." |
| "Session expired" | "פג תוקף החיבור. אנא התחבר מחדש." |

### 3.2 חומרות וטון

| חומרה | טון | דוגמה |
|-------|-----|-------|
| Info | ניטרלי, מסביר | "שינויים לא יישמרו אוטומטית." |
| Success | חיובי, קצר | "הפרטים נשמרו בהצלחה." |
| Warning | זהיר, ספציפי | "לא נשמרת גיבוי בשבוע האחרון." |
| Error | ישיר, מסייע | "לא הצלחנו לטעון את הנתונים. נסה לרענן." |

## 4. Placeholder Text – כללים

| כלל | דוגמה |
|-----|-------|
| placeholder = **דוגמה** לפורמט, לא label | `placeholder="05x-xxx-xxxx"` |
| לא להשתמש ב-placeholder כ-label יחיד | תמיד יש `<label>` |
| placeholder עדין, לא מבלבל | `text-[hsl(--muted-foreground)] opacity-60` |
| תיאור קצר ותמציתי | `"לדוגמה: ישראל ישראלי"` |
| לא לשים הוראות ב-placeholder | הוראות → helper text מתחת לשדה |

## 5. Section Titles & Page Titles – ניסוח

| רמה | דוגמאות |
|-----|---------|
| כותרת דף (H1) | "פרטי מטופל", "מסמכים רפואיים", "לוח בקרה" |
| כותרת סקשן | "פרטים אישיים", "מדדים", "משימות פתוחות" |
| כותרת כרטיס | "לחץ דם", "תרופות", "ביקורים קרובים" |
| Empty state title | "אין עדיין מסמכים", "לא נמצאו תוצאות" |
| Empty state description | "הוסף מסמך ראשון כדי להתחיל לנהל את הרישום הרפואי" |

**כלל ניסוח:** עברית ישירה. אין "כדי לנהל/לצפות/לבצע" – פשוט שם.

## 7. Confirmation Dialogs – ניסוח

כאשר משתמש עושה פעולה בלתי-הפיכה (מחיקה, ניתוק), תמיד dialog אישור:

```
כותרת: "מחיקת [שם הפריט]?"
גוף: "פעולה זו אינה ניתנת לביטול. הנתונים יימחקו לצמיתות."
כפתור אישור (destructive): "כן, מחק"
כפתור ביטול (outline): "לא, חזרה"
```

## 8. Microcopy לפי קונטקסט – MemorAId

### 8.1 Progress קליטה (Profile Completion)

| אחוז | הודעה |
|------|-------|
| 0–25% | "כדי להתחיל – הוסף פרטים בסיסיים" |
| 26–50% | "עוד קצת – השלם את הפרטים הרפואיים" |
| 51–75% | "כמעט שם – בדוק פרטי תרופות ואנשי קשר" |
| 76–99% | "מצוין! רק עוד כמה פרטים לסיום" |
| 100% | "הפרופיל מלא – כל הנתונים מעודכנים" |

### 8.2 Rights Center

| כרטיס | כפתור | תוכן expanded |
|-------|-------|--------------|
| זכות לייעוץ | "פתח מדריך" | bullet list של שלבים לקבלת הזכות |
| זכות לדיור | "פתח מדריך" | מידע על זכאות וקריטריונים |

**כלל:** אין "שלח בקשה" אם אין workflow ברור. רק "פתח מדריך" / "הצג מידע".

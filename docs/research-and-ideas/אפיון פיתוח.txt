Context
הצוות בנה מערכת אדמין ל-MemorAId עם תשתית AI קיימת (ניתוח פרויקט, תכנון פיצ'רים, שאלות ל-AI, לוח מחוונים Intelligence Hub). כעת רוצים להוסיף "מוח" מאחורי האדמין — מערכת Nexus Brief שבה:

מעלים רעיון/פרומפט
המערכת אוספת אינטליגנציה מהרשת (GitHub, Reddit, RSS, Perplexity)
כל מחלקה בחברה (CEO, CTO, CPO, R&D, Design, Product, Security, Legal, Marketing) מבצעת מחקר עצמאי עם AI
התוצאות מתאגדות לניירת (Brief) מובנית לסקירה → אישור / עריכה / דחייה
לאחר אישור: יצירת משימות פיתוח אוטומטית

אין שינויים בצד המשתמש. הכל נמצא בתוך האדמין הקיים.

```
**Pipeline בסדר:**
1. Update status → `'researching'`, emit `start`
2. `gatherProjectData()` → emit `codebase_ready`
3. `gatherWebIntelligence()` → save to `nexusBriefWebSources` → emit `web_intelligence_done`
4. `runAllDepartmentAgents()` (parallel) — לכל מחלקה שמסיימת: save to `nexusBriefDepartments` → emit `department_done`
5. `assembleBrief()` → save to `nexusBriefs.assembledBrief`, status → `'review'` → emit `assembly_done`
6. emit `done` + `res.end()`

**Error recovery:** כשלון מחלקה = `department_error` + ממשיכים. כשלון קריטי = status חזרה ל-`'draft'` + emit `error`.

### nexusAdminRoutes.ts

Sub-router מ mount ב-`index.ts` כ-`/api/admin`:
```
POST   /nexus/briefs              יצירת brief (draft)
GET    /nexus/briefs              רשימה (status filter, pagination)
GET    /nexus/briefs/:id          פירוט מלא + departments + sources
PATCH  /nexus/briefs/:id          עדכון (title, reviewNotes, status, assembledBrief)
DELETE /nexus/briefs/:id          מחיקה (draft/rejected בלבד)

POST   /nexus/briefs/:id/run      הפעל אורקסטרטור → SSE redirect
GET    /nexus/briefs/:id/stream   SSE endpoint (Content-Type: text/event-stream)
POST   /nexus/briefs/:id/approve  אישור → status:'approved'
POST   /nexus/briefs/:id/reject   דחייה + reviewNotes
POST   /nexus/briefs/:id/generate-tasks  יצירת devTasks + sprint

GET    /nexus/models              רשימת מודלים זמינים
GET    /nexus/departments         רשימת מחלקות עם שמות עבריים
```

Rate limit: `aiPostLimiter` על `/run` ו-`/generate-tasks`.

SSE headers חובה:
```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
X-Accel-Buffering: no   ← חובה למניעת nginx buffering
Keepalive: res.write(':keepalive\n\n') כל 15 שניות.

Verification

DB: npm run db:push ← לוודא 3 טבלאות חדשות ב-Supabase Studio
SSE: פתח DevTools > Network > /stream endpoint — לראות event stream חי
End-to-end flow: צור brief → הפעל → ראה מחלקות מסתיימות בזו אחר זו → אשר → בדוק devTasks שנוצרו בטבלה
Trust Score: GitHub source עם stars > 1000 צריך לקבל score > 60
Error resilience: Kill PERPLEXITY_API_KEY temp → וודא שאר המחלקות ממשיכות
Rate limit: שלח 11 בקשות סימולטניות → לוודא 429 ב-11ה

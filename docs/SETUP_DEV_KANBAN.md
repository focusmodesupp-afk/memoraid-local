# הגדרת Dev Kanban System - מדריך מהיר

## ⚡ התקנה מהירה (3 שלבים)

### שלב 1: הפעל את השרת
```bash
npm run dev
```

### שלב 2: צור את הטבלאות
פתח דפדפן וגש ל:
```
http://localhost:5000/api/health/create-dev-kanban-tables
```

או דרך PowerShell:
```powershell
Invoke-WebRequest -Uri "http://localhost:5000/api/health/create-dev-kanban-tables" -Method POST
```

תקבל תשובה:
```json
{
  "ok": true,
  "message": "Dev Kanban tables created successfully with 6 default columns!"
}
```

### שלב 3: גש לקנבאן
```
http://localhost:5000/admin/dev/kanban
```

## ✅ מה נוצר?

### 6 עמודות ברירת מחדל:
1. **Backlog** (אפור) - משימות עתידיות
2. **To Do** (כחול) - משימות מתוכננות
3. **In Progress** (כתום) - בעבודה כרגע
4. **QA** (סגול) - בבדיקה
5. **Support** (תכלת) - תמיכה/עזרה נדרשת
6. **Done** (ירוק) - הושלם

### 3 טבלאות במסד הנתונים:
- `dev_columns` - עמודות הקנבאן
- `dev_tasks` - משימות הפיתוח
- `dev_comments` - תגובות למשימות

## 📝 שימוש יומיומי

### הוספת משימה חדשה:
1. לחץ על כפתור "+ משימה חדשה"
2. הזן כותרת
3. בחר עמודה
4. לחץ "צור"

### עריכת משימה:
1. לחץ על המשימה
2. ערוך בטאבים:
   - **פרטים**: כותרת, תיאור, עדיפות, קטגוריה
   - **תכנון**: אחראי, תאריך יעד, הערכת שעות, תגיות
   - **פעילות**: תגובות ודיונים

### הוספת עמודה חדשה:
1. לחץ על כפתור "עמודות"
2. לחץ "+ הוסף עמודה"
3. הזן שם ובחר צבע
4. לחץ ✓

### העברת משימה:
פשוט גרור את המשימה לעמודה אחרת!

## 🔧 Troubleshooting

### הקנבאן ריק?
הטבלאות עדיין לא נוצרו. הרץ:
```
http://localhost:5000/api/health/create-dev-kanban-tables
```

### שגיאת "Table not found"?
אותו פתרון - צור את הטבלאות.

### השרת לא עובד?
```bash
npm run dev
```

### רוצה לאפס הכל?
מחק את הטבלאות ב-Drizzle Studio:
```bash
npm run db:studio
```

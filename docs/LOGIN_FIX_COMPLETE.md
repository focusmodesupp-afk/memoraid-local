# תיקון לוגין אדמין – סיכום מלא

## מה תוקן (לפי דו"ח ניתוח)

1. **health/tables הועבר ל-index.ts**  
   כעת הבדיקה לא תלויה ב-adminRoutes. גם אם יש שגיאה בייבוא adminRoutes, `health/tables` עדיין יעבוד ויציג הודעת תיקון.

2. **חזרה 503 במקום 500**  
   כשאין טבלאות אדמין (42P01), auth/me ו-auth/login מחזירים 503 במקום 500, כדי להפריד בין "שרת לא זמין" ל"טבלאות חסרות".

3. **סקריפט fix:login**  
   פקודה אחת: `npm run fix:login` – בודקת חיבור, יוצרת טבלאות ומשתמש אדמין.

## אם עדיין לא עובד

1. **בדיקת חיבור DB:**
   ```bash
   node scripts/test-db-connection.mjs
   ```

2. **בדיקת טרמינל השרת** – חפש הודעות:
   - `health/tables DB error: 42P01` → טבלאות חסרות
   - `Admin login error [code=42P01]` → טבלאות חסרות
   - `ECONNREFUSED` → אין חיבור ל-DB

3. **בדיקת .env** – `DATABASE_URL` חייב להיות תקין. אם הסיסמה מכילה `@`, להחליף ל-`%40`.
